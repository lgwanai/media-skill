# Media Skill - AI 视频创作与分析工作站

## 📖 项目介绍
Media Skill 是一款全能型的 AI 视频创作与分析工作站，深度融合多模态大模型（如小米 Omini）与自然语言处理（NLP）技术。我们的长期定位涵盖五大场景：**视频智能剪辑**（已开放）、**基于主题的片段提取**（已开放）、**文生视频**、**视频多模态分析**、**高精度字幕识别**。

当前版本主要聚焦于**视频智能剪辑**与**长视频主题提取**工作流。通过自动识别并精准剔除视频中的无声片段、语气词（如“额”、“啊”）、重复表达及磕巴口误，帮助创作者大幅降低后期人工剪辑的时间成本，快速输出紧凑、流畅的成片。同时，利用文本大模型的能力，创作者可以输入任意主题描述，AI 会自动从长视频中定位并提取所有相关的片段，并在保证语义连贯性的前提下输出最终合集。

## ✨ 项目亮点 (当前版本)
- **多级混合智能剪辑架构**：结合规则过滤（静音检测）、大模型语义分析（剔除废话）与多模态精修，支持视频的“部分剔除（Partial Trim）”与跨片段精准裁剪。
- **基于主题的智能提取**：支持对长视频进行“大海捞针”式的主题检索。不仅能精准命中相关片段，还会自动保留上下文以确保语义连贯（如 2 和 4 相关，自动保留 2、3、4），对于超长文本自动进行无损切块分析。
- **强大的多模态分析能力**：深度接入小米官方 `mimo-v2-omni` 模型，不仅能“看懂”画面，更能深度理解音视频的深层对应关系与情绪表达，完成高难度瑕疵判定。
- **云端存储无缝集成**：内置腾讯云 COS 自动上传机制，提升大模型分析时的调用效率与稳定性。
- **高可用与防呆设计**：具备完善的 API 错误降级处理（如遇到 Omini API 402 错误回退至规则过滤），并提供一键式打包发布脚本。

## � 项目计划 (Roadmap)
以下核心功能与场景正在积极规划与研发中，将在未来版本陆续开放：
1. **文生视频**：支持输入小说、脚本，直接驱动生成高质量视频内容，实现“一键出片”。打通从“文本”到“视频”，再到“后期剪辑”的完整智能创作闭环。
2. **视频多模态分析**：深度洞察热点爆款视频，解析画面呈现与文案节奏的映射与逻辑关系，拆解爆款基因，为内容创作提供高阶指导。
3. **高精度字幕识别 (说话人分离)**：超越传统语音识别，提供毫秒级精确的语音转文字服务，支持区分不同说话人（Speaker Diarization），让多人访谈、播客处理游刃有余。

## 💎 核心价值

## 🎯 使用场景
- **口播视频/Vlog 后期**：自动清理 UP 主录制时的忘词卡壳、语气词和重复表达。
- **教学/教程视频制作**：去除讲课过程中的重复解释，输出干练的教学视频。
- **小说/短剧视频化 (规划中)**：输入文本小说或分镜脚本，直接生成配音与画面结合的视频流。
- **爆款短视频拆解 (规划中)**：输入热门视频，提取并分析其画面呈现规律与文案节奏，指导后续创作。
- **播客/访谈录像精简与字幕 (规划中)**：快速剔除无声等待时间，精确区分多位嘉宾发言，一键生成带有角色标签的精确字幕。

## ⚙️ 工作方式
项目核心剪辑流程分为五个独立且递进的步骤：
1. **Step 1 - 虚拟硬切片 (Metadata Slicing)**：摒弃传统产生大量临时文件的物理切片方式，直接解析 FunASR 提供的时间戳，在内存中计算并记录各片段（带 300ms 连贯缓冲）的绝对时间轴元数据，实现极速处理与零磁盘 I/O。
2. **Step 2 - LLM 语义分析 (Semantic Analysis)**：一次性将所有台词交由文本大模型进行结构化分析，精准识别语气词、无意义废话，通过追溯上下文判断并标记语义重复，同时要求大模型直接提取出需剔除的“废话原句”。
3. **Step 3 - 字级时间戳精细匹配 (Precise Trimming)**：对于大模型标记出的局部磕巴或重复（废话原句），算法会与 FunASR 提供的字符级/词级时间戳进行精准碰撞，计算出毫秒级的绝对剔除区间，无需依赖视觉大模型盲猜时间。
4. **Step 4 - 非语言片段智能保留 (Silence Analysis)**：(可通过 `ENABLE_SILENCE_ANALYSIS` 配置开启) 自动计算语言片段之间的空隙。针对这些纯画面片段，系统会截取并压缩上传至腾讯云 COS，交由小米 Omini 视觉大模型判定其是否包含有意义的鼠标操作、人物动作或运镜，从而决定保留或删除。
5. **Step 5 - 终极无损合并 (Final Merge)**：基于上述步骤生成的精确保留与挖空时间轴，利用 FFmpeg `filter_complex` 一次性从原视频上进行毫秒级精准裁剪与拼接，确保输出视频画质与原视频 100% 一致且无任何倒退、错位问题。

## 🚀 安装与使用

### 1. 环境准备
- **Python 3.8+**
- **FFmpeg** (必须安装并配置到系统环境变量中)

### 2. 安装依赖
```bash
pip install -r requirements.txt
```
*(注：项目核心依赖包含 `cos-python-sdk-v5`, `openai`, `funasr` 等)*

### 3. 配置文件准备
复制示例配置文件并填入真实的 API 密钥：
```bash
cp config.example.txt config.txt
```
在 `config.txt` 中需要配置：
- 文本大模型 API 密钥
- 小米 Omini API 密钥 (`MIMO_API_KEY`)
- 腾讯云 COS 配置 (`COS_SECRET_ID`, `COS_SECRET_KEY`, `COS_REGION`, `COS_BUCKET_NAME`)
- `ENABLE_SILENCE_ANALYSIS` (默认为 false，设为 true 可开启纯画面智能判定保留功能)

### 4. 运行剪辑
首先确保使用 `transcribe.py` 生成了带精确时间戳的转写 JSON 文件：
```bash
python scripts/transcribe.py <待剪辑视频路径.mp4>
```
然后将转写 JSON 和原视频作为参数传入智能剪辑脚本：
```bash
python scripts/clip_video.py output/<视频名>/transcription.json <待剪辑视频路径.mp4>
```
处理过程中的中间产物和最终结果将保存在 `output/` 目录下。

### 5. 项目打包
如需将项目打包分发，请运行自带的打包脚本。生成的纯净发布包（排除敏感配置与缓存）会自动存入 `dist/` 目录：
```bash
./package.sh
```

## ❓ 常见问题 (FAQ)

**Q: 遇到 Omini API 报 `402 Payment Required` 错误怎么办？**
A: 脚本已内置降级逻辑。当多模态大模型额度耗尽或调用失败时，系统会自动回退到“基于规则和文本大模型”的语义过滤模式，依然能有效处理大部分语气词和无声片段，保证流程不中断。

**Q: 为什么之前版本生成的 SRT 或 TXT 字幕文件为空？**
A: 这是早期版本 API 返回字段不匹配（`sentences` 与 `sentence_info` 混淆）导致的，该问题已在当前最新版本中彻底修复，请确保代码已更新。

**Q: 我的 COS 密钥和大模型 API 密钥安全吗？**
A: 安全。`config.txt` 已被加入 `.gitignore`，不会被提交到 Git 仓库。同时，自带的 `./package.sh` 打包脚本也严格排除了这些敏感文件。请仅分享或提交 `config.example.txt`。

**Q: 遇到模块导入错误 `cannot import name 'CosS3Client'`？**
A: 请检查你的 Python 环境，卸载旧版的 `qcloud-cos-python3` 包，并确保安装的是官方推荐的 `cos-python-sdk-v5`。
