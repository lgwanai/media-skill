# Media Skill: 多媒体生成、识别与智能剪辑一体化工作流

本技能旨在帮助用户通过一系列自动化、智能化的工作流，对音视频文件进行高精度识别、时间轴提取以及基于语义和多模态理解的智能剪辑。

## 核心能力
1. **高精度音视频转写 (独立功能)**：将音视频转换为文字，附带精确到毫秒级的时间戳。支持一键生成区分说话人的专业级字幕。
2. **多说话人分离 (Speaker Diarization)**：准确标记不同说话人，处理重叠发言，适用于播客、会议访谈等场景。
3. **基于主题的内容提取 (独立功能)**：用户输入主题描述，系统自动进行长视频/音频分析，并基于语义连贯性，智能提取与合并所有相关片段。
4. **多模态场景理解**：直接对视频画面和声音进行意图理解。
5. **四步智能剪辑**：从初步时间戳打底，到文字语义精剪，再到多模态深度精剪，最后综合成片合并。

## 系统配置要求
系统依赖外部 `config.txt` 文件进行配置，处理任何任务前，请先确保当前目录下存在 `config.txt` 并读取其内容：
- `MODEL_DIR`: 模型存放目录
- `OMINI_URL`: Omini 大模型 URL
- `OMINI_API_KEY`: Omini API Key
- `OMINI_MODEL_NAME`: Omini 模型名称
- 腾讯云 COS 配置（用于多模态剪辑的切片上传）

## 执行工作流指南

### 1. 环境与依赖检查
在开始处理之前，检查系统是否安装了 `ffmpeg` 并在环境变量中。同时确保 Python 环境已安装 `requirements.txt` 中的 `funasr`, `torch`, `modelscope`, `cos-python-sdk-v5`, `openai` 等依赖。如果缺失，请指导用户或自动通过终端安装。

### 2. 独立功能：毫秒级字幕识别与说话人分离
当用户仅要求“提取字幕”、“识别语音”、“区分说话人”时，请作为独立功能调用以下命令：
```bash
python scripts/transcribe.py <音视频文件路径>
```
**功能特性**：
- 使用 `ffmpeg` 提取音频并重采样至 16kHz。
- 调用 FunASR Pipeline (`FSMN-VAD + Paraformer + CAM++`) 进行高精度识别与说话人分离。
- **输出产物**：在 `output/` 目录下生成包含**毫秒级时间戳和说话人标签**的 SRT 字幕文件 (`transcription.srt`)、TXT 纯文本 (`transcription.txt`)，以及包含详细词级/句级数据的 JSON 文件 (`transcription.json`)。

### 3. 独立功能：基于主题的长视频/音频片段提取
当用户要求“根据描述提取片段”、“找一下关于xxx的部分”时，请作为独立功能调用以下命令：
```bash
python scripts/extract_by_theme.py <音视频文件路径> "<主题描述>"
```
**功能特性**：
- 先进行毫秒级转字幕。
- 基于大模型进行带上下文分析：如果片段 2 和 4 与主题相关，为了保证语义连贯，会自动保留 2、3、4。如果中间跨度过大（如 2 和 10），则分别提取。
- **自动分块**：对于长于 10000 字符的字幕，脚本会自动按最大安全长度切块后交给 LLM 处理，并最终进行精准的时间轴合并。
- **输出产物**：提取的最终视频会保存为 `output/theme_extract/theme_final.mp4`，中间切片保存在 `clips` 目录下。

### 4. 智能视频剪辑（4步法）
当用户请求“智能剪辑”或“精剪”时，严格执行以下 4 步策略：

#### 第一步：初步识别（时间戳打底）
- 对视频进行转写初步识别，**只识别，不剪辑**。
- 将视频中的“空白”、“气口”等无效时间点准确记录为 JSON 文件（例如 `step1_silence.json`）。

#### 第二步：文字语义精剪（切片摘取）
- 根据第一步转写的文字进行语义分析，找出语气词过多、重复、语句不通顺的片段。
- 依据时间轴使用 `ffmpeg` 对视频进行切片。
- 将这些切片存入临时文件夹，并在 JSON 中记录这些被摘取片段的时间轴（例如 `step2_semantic_clips.json`）。

#### 第三步：多模态深度精剪（降本增效）
- 对临时文件夹中的切片视频进行压缩：使用 `ffmpeg` 将其转为 540p、30帧的轻量视频，以降低大模型 Token 消耗。
- 通过读取 `config.txt` 中的 Omini 大模型配置，调用多模态大模型 API 对这些压缩切片进行识别和理解。
- 再次进行精剪判定，精准剔除重复口语、气口、无意义内容。
- **不直接物理剪辑**，将需要剔除或保留的时间点更新至 JSON 文件（例如 `step3_multimodal_filter.json`）。

#### 第四步：综合成片合并
- 结合前三步产出的所有 JSON 数据，计算并生成最终的“有效时间 JSON”。
- 根据该“有效时间 JSON”，使用 `ffmpeg` 对**原始视频**进行精确切割与合并，生成最终的成片视频素材。

## 异常处理与性能
- 并发推理时需注意缓存冲突（可通过队列或单线程处理避免）。
- 遇到解码错误应跳过该片段并记录日志，不可中断整体长视频处理。
